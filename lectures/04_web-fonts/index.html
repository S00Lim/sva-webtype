Loading Fonts

Fonts are a type of file the webpage needs to load along with the .html files to display a webpage. These fonts may be stored in different places:

  • Core web fonts (Visitor has the font files)
  • Self-hosted (You have the font files)
  • Distributor-hosted (ie. Type Network)
  • Subscription-based (Adobe, Google)

Self-hosted

Use the @font-face rule to link to typefaces in your website folder.

In your CSS file: 
					
@font-face{
	font-family: 'My Font';
	src: url('fonts/FontName-Medium.woff2') format('woff2');
	font-weight: 400;
}
@font-face{
	font-family: 'My Font';
	src: url('fonts/FontName-MediumItalic.woff2') format('woff2');
	font-style: italic;
	font-weight: 400;
}

body {
	font-family: 'My Font'; 
	/*this name should match the font-face rule you set up*/
}

Variable Fonts

Variable fonts are a new format (c. 2016) that allows multiple styles of a font to be embedded into a single font. Browsers were the first to adopt support for this new font format. More on expressive advantages of variable fonts.

Variable fonts

Including variable fonts works similarly, where multiple values are accepted for font-weight or font-stretch if applicable to define the possible range within the font. The format is updated to woff2-variations.

The font-variation-settings is the CSS property for accessing the variable font settings. It is followed by comma-separated assignments for each axis, in the format 'key' numericvalue.

In your CSS file: 
					
@font-face{
	font-family: 'Font Name VF';
	src: url('fonts/FontName-VF-Roman-Trial.woff2') format('woff2-variations');
	font-weight: 200 700;
}

body{
	font-family: 'Font Name Font'; 						
	font-variation-settings: 'wght' 400;
}

h1{
	font-family: 'Another Multi-axis Variable Font';							
	font-variation-settings: 'wght' 300, 'wdth' 120, 'XHGT' 100; 
}

Standard Axis Tags

A variable font often contains standard axes based on how typeface families are often defined, such as variations in weight, width, and slant. These are denoted in a 4-letter, lowercase axis tag. Custom variable axis names defined by the type designer are set in ALLCAPS.

Axis Tag Description Valid Range CSS Equivalent
wght weight 1—1000 font-weight
wdth width >0, % of normal width font-stretch
opsz optical sizing >0, point sizes
ital italicization 0—1 font-style
slnt slant -90—90 font-style

Understanding the Axes of a Font

Use a font like Font Goggles or an online tool like Wakami Fondue to understand the variable range of your typeface. (For Adobe/Google hosted fonts, check out the font detail pages for this info.)

Open Type Features

Open Type features provide ways to access embedded functionality in your typefaces that help you refine your typography by providing alternate forms. Depending on the genre of the font, these might include some from the list below. The general syntax font-feature-settings: 'featuretag' on/off; or 1/0 (0 meaning off, 1 meaning on) provides general access to the features; some features may have a more specific CSS properties on modern browsers. Depending on how the typeface designer set up the font, some features may be on by default.

Stylistic Alternates

Features can also handle alternative glyphs contained within the typeface.

*The variants assigned to stylistic sets are up to the type designer—there are no standard mappings. A font can have up to 20 ss alternates.

†Localized forms need to be combined with the lang="xx" attribute on the element, where xx is the 2-letter ISO language code.

Multiple feature properties can be applied by comma-separating the different feature activations. If you set another font-feature-settings property below another, it will cancel the first setting, so the features must always be combined into a single line. For example, font-feature-settings: 'liga' 1, 'ss01' 1; will turn on both ligatures and Stylistic Set 01 (i.e. a simpified a) to the element.
