Using Github

Getting started with a publishing workflow

SVA Section 3 24/FA DSD-3662-A
Web Type Specimens

Set up Github

  1. Sign up for an account at GitHub.
  2. Download the Github Desktop application and install it on your computer.
  3. Open Github Desktop and sign-in with your account.

Clone the Class Repository

This repository contains all of the demos for the class in the demos folder.

  1. Log into github.com.
  2. Go to this repository on Github and click the green Clone button and choose Open in Desktop.
  3. When you press this button, it should launch the Github desktop application you downloaded before.(If it doesn’t happen, open the application manually.)
  4. Select a place on your computer where you want your copy of the repository to live. It might be somewhere like /Users/motsuka/Sites.
  5. Every week, make sure to click on the Fetch origin button on the top right. This syncs up your local copy the repository the folder with the latest code.

Create your own repository

  1. Open your browser to get back to github.com, and create a new repository using +New button on the leftside of your GitHub homepage.
  2. Name your repository username.github.io. For example, my username is marieotsuka, so my repository should be named marieotsuka.github.io. When you create the repository, check the box to initiate with a .README file. This repository will be the home for all of your assignments and exercises for this class. (If you already have a Github account and a repository with that name, you might make a different repository name instead. In this case, be careful of root-relative links, because your website will live in username.github.io/project-name)

Clone your own repository

  1. Now, clone your own new repository, in the same way that you cloned the class repository.
  2. After clicking Open in Desktop, click the Choose button and navigate to the folder where you plan to save your work for this class, then press the Open button. This is where the local version of your project will live, and your working directory for all website files for this class.
  3. You should now have a folder called username.github.io on your computer. Make sure to save all of your work into this folder.

Create your first webpage!

  1. Navigate to the class repository on your Finder.
  2. Copy the contents of demos/00-starterkit folder into your new repository (the index.html and style.css file.)
  3. Paste these files into your own repository folder. Do not copy the entire folder, instead, select only the files within the folder and copy them to your own repository folder. The index.html file should be located directly within your repository (what we call your root) not in a subfolder.

Open your HTML file in the Browser

  1. Double-click on the index.html file to open it in your Browser.
  2. Note: if your default browser is Safari, I recommend changing this to Chrome or Firefox at this time for easier access to browser tools.
  3. You’re now seeing your own very simple website!

This is a local preview of your website, only accessible to you. Notice that the url is something specific to your computer, i.e. file:///Users/motsuka/Sites/sva-webtype/demos/00-starterkit/index.html

Commit and Push to your repository

Uploading your local files on your computer to the remote server—in our case, hosted on Github—requires 2 steps: commit and push. Don’t forget to push after committing your code!

Commit

This will add and save any changes since your last commit to the local repository. It is a way to save a “snapshot” of the code of your website — a version to go back to, as reference. This is the “Git” version-control program that is part of Github. Note: you must fill out the Summary field with a description of your update, to commit files.

Push origin

This will send your committed changes to a remote repository. This second step is necessary for to push your code to the Github servers, and for others to see the changes on your website.

Confirm your Changes

  1. Confirm you see changes made to your file in the Changes Tab on the Github app. Fill in a brief description of the change you made in the Summary field — this is a required field, and a note for your future self.
  2. Press Commit button and then Press Push Origin in the top right tab.
  3. Check the remote files by navigating to github.com/yourusername. Verify that your changes were pushed to the master repository.

Publish your website

In your GitHub.com repository page, find the “Settings” tab. Select Github Pages on the sidebar, and press the Select Source and choose master from the dropdown. This is the source files for your webpage.

After you click save, click the link listed in the same section, which should be username.github.io to view your website that should say “Hello World”. You can now upload any page and publish it to the web!

Your Live Site

Your live webpage will take on the name of your repository. If your repository is named username.github.io, your website url will be https://marieotsuka.github.io/. (If you used a diffferent repository name, it will be at https://marieotsuka.github.io/repositoryname.)

It is important to confirm that everything is working on the live version of your website. This is the URL you should use to share your work and submit assignments. Do not use your local preview URL beginning with file:. This won’t be accessible to anyone else.

Note: It may take a couple of minutes for the live website to reflect updates after pushing your changes to Github. Make sure you also do a hard refresh of your site to clear any cache. (Keyboard shortcut is Shift + command + R).
